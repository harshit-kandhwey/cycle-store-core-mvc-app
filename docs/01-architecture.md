# Architecture

## Tiers

The store is a classic three-tier web application:

```
Browser ──HTTPS──▶  Web tier (nginx)  ──HTTP──▶  App tier (.NET/Kestrel)  ──TDS──▶  DB tier (SQL Server)
                    TLS termination,             ASP.NET Core MVC on :5000          CYCLE_STORE, schema
                    reverse proxy, :80→:443      EF Core 8                          "Production", :1433
```

Two supported physical topologies (see [02-deployment](02-deployment.md)):

- **Scenario 1** — Web + App collapsed on one VM (nginx proxies to `127.0.0.1:5000`), DB on a second VM.
- **Scenario 2** — Web, App, DB each on their own VM.

The app code is identical in both; only `cyclestore.conf` differs.

## Request flow

1. nginx terminates TLS and reverse-proxies to the app tier (`APP_HOST:APP_PORT`),
   forwarding `Host` and `X-Forwarded-*`.
2. Kestrel runs the MVC app. Routing: `{controller=Home}/{action=Index}/{id?}`.
3. Controllers call the service layer (`IProductService`, `ICategoryService`),
   which issues EF Core queries against SQL Server.
4. What the storefront shows is narrowed **in the app** by `CatalogCuration`
   (an allow-list of subcategories + product numbers), so the DB can hold more
   than the store displays.

## Key components (app tier)

| Area          | Files                                                                                                                       |
| ------------- | --------------------------------------------------------------------------------------------------------------------------- |
| Controllers   | `Controllers/HomeController`, `ProductsController`, `CartController`                                                        |
| Services      | `Service/Interface` + `Service/Implementation` (EF Core data access)                                                        |
| Domain/model  | `Models/Product`, `ProductCategory`, `ProductSubcategory`, `CYCLE_STOREContext`                                             |
| Catalog logic | `Models/CatalogCuration` (what shows), `CatalogImages` (photo resolution), `CatalogContent` (brand/description/rating/sale) |
| Cart          | `Models/CartStore` (session), `CartViewModel`                                                                               |
| Views         | `Views/Products`, `Views/Cart`, `Views/Shared/_Layout`                                                                      |
| Assets        | `wwwroot/css/cyclestore.css`, `wwwroot/js/cyclestore.js`, `wwwroot/Images/catalog`                                          |

## Runtime configuration & security

- Connection string, `AllowedHosts`, URLs, and environment come **only** from
  configuration (the `cyclestore.env` file generated by `cyclestore-apply`); no
  secrets live in source.
- Security middleware sets `X-Content-Type-Options`, `X-Frame-Options`,
  `Referrer-Policy`, and a **nonce-based Content-Security-Policy**.
- `.avif` is registered with the static-file middleware (most product photos are AVIF).

## Tech stack

ASP.NET Core MVC (.NET 8), EF Core 8, SQL Server 2019, Razor, vanilla CSS/JS
(no front-end build), nginx, systemd.
